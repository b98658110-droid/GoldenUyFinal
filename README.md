# GoldenUy — платформа недвижимости по Узбекистану

## Запуск проекта

1. Установить зависимости:
   ```
   npm install
   ```

2. Открыть `.env` и указать свои реальные данные подключения к PostgreSQL
   (PGHOST, PGPORT, PGUSER, PGPASSWORD, PGDATABASE) — те же, что использовались
   в psql при создании базы `goldenuy`.

3. Проверить, что в базе есть таблицы `users`, `properties`, `favorites`, `cart`
   с полями, описанными в `schema.sql`. Если каких-то полей не хватает —
   создать их (`schema.sql` использует `CREATE TABLE IF NOT EXISTS`, поэтому
   его можно безопасно выполнить повторно, существующие таблицы не пострадают).

4. Создать первого администратора. Самый простой способ — зарегистрироваться
   обычным способом через `/register`, а затем в psql выполнить:
   ```sql
   UPDATE users SET role = 'admin' WHERE email = 'b98658110@gmail.com';
   ```

5. Запустить сервер:
   ```
   npm start
   ```
   или в режиме разработки с автоперезапуском:
   ```
   npm run dev
   ```

6. Открыть в браузере: http://localhost:3000

## Структура

- `app.js` — точка входа, настройка Express, сессий, маршрутов
- `db.js` — подключение к PostgreSQL
- `routes/auth.js` — регистрация, вход, выход
- `routes/property.js` — главная страница, объект, избранное, корзина
- `routes/admin.js` — админ-панель (CRUD объектов недвижимости, Multer)
- `middleware/auth.js` — проверка авторизации (requireAuth, requireAdmin)
- `views/` — все EJS-шаблоны
- `public/` — статика: CSS, изображения, клиентский JS
- `uploads/` — папка для загруженных фотографий объектов

## Важно

- Пароли хранятся в виде bcrypt-хэша, никогда в открытом виде.
- Все SQL-запросы параметризованы ($1, $2...) — защита от SQL-инъекций.
- Сессии хранятся в самой PostgreSQL (таблица `session`, создаётся
  автоматически при первом запуске) — пользователи не разлогиниваются
  при перезапуске сервера.
- При удалении объекта недвижимости связанные записи в `favorites` и `cart`
  удаляются автоматически благодаря `ON DELETE CASCADE` в схеме БД.
